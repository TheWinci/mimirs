# Glossary

## AST

**Abstract Syntax Tree** -- A tree representation of source code structure. [Tree-sitter](#tree-sitter) parses source files into ASTs, which the indexer uses to produce semantic [chunks](#chunk).

## BM25

**Best Matching 25** -- A probabilistic text ranking algorithm. mimirs uses BM25 via SQLite's [FTS5](#fts5) extension for keyword matching, which is combined with vector similarity in [hybrid search](#hybrid-search).

## Chunk

A semantically meaningful fragment of a source file stored alongside its [embedding](#embedding). A chunk might be a single function, a class, a markdown section, or another logical unit identified by the [AST](#ast)-based splitter. Chunks are the atomic unit of search. See the [Chunk entity](entities/chunk.md).

## ChunkResult

A search result that includes the full chunk content and exact line ranges (e.g., `src/db.ts:42-67`). Returned by the `read_relevant` tool and the `searchChunks()` function.

## DedupedResult

A file-level search result created by merging the best [ChunkResult](#chunkresult) snippets from the same file. Returned by the `search` tool and the `search()` function.

## Embedding

A 384-dimensional `Float32Array` vector that captures the semantic meaning of a text [chunk](#chunk). Generated by the all-MiniLM-L6-v2 ONNX model via Transformers.js and stored in [sqlite-vec](#sqlite-vec) virtual tables for similarity search.

## FTS5

SQLite's full-text search extension. Provides the [BM25](#bm25) keyword matching component of [hybrid search](#hybrid-search). Each chunk's text is indexed in an FTS5 table alongside the vector index.

## Hybrid Search

The primary search strategy. Combines [embedding](#embedding)-based vector similarity with [BM25](#bm25) text search, blending their scores with configurable weighting (default 70% vector, 30% BM25). This yields better results than either approach alone -- vectors capture semantic meaning while BM25 catches exact keyword matches. See the [Hybrid Search entity](entities/hybrid-search.md).

## MCP

**Model Context Protocol** -- A standard for exposing tools to AI agents. mimirs runs as an MCP server, presenting tools like `search`, `read_relevant`, and `project_map` to IDEs that support the protocol (Claude Code, Cursor, Windsurf, VS Code Copilot, JetBrains).

## RagConfig

The Zod-validated configuration type that controls all tunable behavior (chunk sizes, search weights, embedding model, etc.). Stored in `.mimirs/config.json` and auto-populated with defaults on `init`. See the [RagConfig entity](entities/rag-config.md).

## RagDB

The central database facade class. Wraps a SQLite database handle and coordinates delegate modules for indexing, search, graph operations, conversations, checkpoints, annotations, and analytics. Most tools and CLI commands operate through a RagDB instance. See the [RagDB entity](entities/rag-db.md).

## sqlite-vec

A SQLite extension that enables vector similarity search via [vec0](#vec0) virtual tables. Stores [embeddings](#embedding) as `Float32` arrays and supports approximate nearest-neighbor queries.

## Tree-sitter

An incremental parser framework that produces [ASTs](#ast) for source code. mimirs uses tree-sitter (via `@winci/bun-chunk`) to parse 30+ languages and split files into semantic [chunks](#chunk).

## vec0

The [sqlite-vec](#sqlite-vec) virtual table type. Each row stores a `Float32` vector alongside metadata, enabling similarity search queries against the [embedding](#embedding) index.

## WAL

**Write-Ahead Logging** -- A SQLite journal mode that allows concurrent readers while a single writer is active. mimirs uses WAL mode so the MCP server can handle search queries without blocking the indexer.
